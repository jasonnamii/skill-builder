# 신규 스킬 템플릿

새 스킬 생성 시 이 스켈레톤에서 시작. 필수 구조를 빠뜨리지 않기 위한 가이드.

---

## SKILL.md 스켈레톤

```markdown
---
name: {skill-name}
description: |
  {핵심 용도 1문장. 무엇을 하는 스킬인지.}
  P1: {고유 명사 키워드 5개+, 쉼표 구분}.
  P2: {동사 패턴 한+영 2개+}.
  P3: {영어 기술용어 2개+}.
  P4: {상황적 표현 0-2개. 선택}.
  P5: {출력 형태 1개+}.
  NOT: {제외 키워드}(→{대체 스킬}).
---

# {Skill Name}

{스킬의 핵심 가치 1줄}
```

## 필수 체크리스트

| 항목 | 근거 |
|------|------|
| YAML frontmatter 있음 | validate.py 파싱 필수 |
| name: kebab-case | 설치 시 폴더명으로 사용 |
| description ≤500자 | validate.py 하드 리밋 |
| P1 5개+, P2 2개+, P3 2개+, P5 1개+, NOT 필수 | 트리거 최소 요건 |
| Gotchas 섹션 있음 | Lean 원칙: Gotchas 필수 |
| SKILL.md ≤10KB | 토큰 예산 |
| "왜" 있음 | 이유 없는 지시 → LLM이 무시 |
| 예시 1개+ | 없으면 출력 자의적 |

## 폴더 구조

```
{skill-name}/
├── SKILL.md          (필수)
├── references/       (5KB+ 시 분리)
├── scripts/          (결정적·반복 작업)
└── assets/           (출력용 파일)
```

## NOT 설계 팁

NOT은 "이 요청은 다른 스킬이 더 적합"을 명시하는 라우팅.
기존 스킬 P1 키워드와 겹치는 게 있으면 반드시 NOT에 추가.
형식: `NOT: {키워드}(→{스킬명})`
