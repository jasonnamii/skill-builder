# NO_WORK_LABEL — 산출물 스킬 강제 주입 블록

**목적:** 모든 문서·산출물 생성 스킬이 "작업 라벨"(C:E:W:, Y2, 4축 등 외부인이 사전 없이 못 읽는 내부 약어)을 산출물·대화에 노출하지 않도록 강제.

**적용 대상:** P5에 파일 확장자(.md/.html/.docx/.pptx/.xlsx/.pdf)가 명시된 모든 신규/수정 스킬, 또는 "산출물·보고서·기획안·제안서·문서·리포트" 등을 본문이 직접 생성하는 스킬.

**적용 위치:** 대상 스킬 SKILL.md의 흐름 블록 직후, 또는 "절대 규칙" 첫 항목으로 아래 블록을 **그대로** 삽입. 줄바꿈·공백·기호 변형 금지.

---

## 삽입 블록 (verbatim — 변형 금지)

```
[NO_WORK_LABEL] §STYLE [CONTENT]
RULE      : 산출물·대화 = 인간 언어. 작업 라벨 ZERO.
            (1만 페이지 1단어 = FAIL)
판정      : "이 단어, 이 대화 밖 사람이 사전 없이 읽을 수 있나?"
            NO → 작업 라벨 → 금지
ALLOW     : 업계 전문용어 (BEP·KPI·MECE·MVP 등) · 고유명사 · 법조문
CONVERT   : 라벨 발견 → 실명·평문 풀어쓰기
            예) Y2 → "2년차" / 4축 → 실제 4개 이름 / C:E:W: → 평문
SELF_CHECK: 산출 직전 자체 스캔. 1개라도 발견 = 차단·재작성
```

---

## 자체 스캔 체크리스트 (산출 직전 게이트)

| # | 체크 | 행동 |
|---|------|------|
| 1 | 한글자·두글자 영문대문자 약어 (Y2·V3·C:·E:·W: 등) | 풀어쓰기 |
| 2 | "N축·N계열·N층·N블록" 등 수치+추상명사 | 실명 N개 나열 |
| 3 | 영문 코드형 라벨 (DOC_TYPE·CLAIM_EVIDENCE·CEW 등) | 한글 평문으로 |
| 4 | 콜론 접두 라벨 (C:·CTA:·HOOK: 등 본문에 노출) | 일반 문장으로 |
| 5 | 업계 표준(BEP·KPI·MECE·MVP·OKR·ROAS 등) | 통과 (ALLOW) |
| 6 | 고유명사·법조문(상법 제340조의2 등) | 통과 (ALLOW) |

**1개라도 발견 → 산출 차단 → 재작성.**

---

## 적용 예시

**BAD (작업 라벨 노출):**
> Y2 매출은 BEP 대비 130%. C:E:W: 구조로 4축 분석.

**GOOD (평문):**
> 2년차 매출은 손익분기점(BEP) 대비 130%. 주장-근거-논거 구조로 시장·제품·조직·재무 4개 관점 분석.

---

## 미적용 = FAIL 사유

- 사용자(고객·외부 독자)가 산출물을 사전 없이 읽지 못함
- 내부 약어 = 작성자 편의, 독자 비용 전가
- 1만 페이지 중 1단어 누출도 FAIL (확률 게이트 아님, 결정적 게이트)
